    # Overview

    单个计算机 ==> 多个计算机通过集线器连接 ==> 形成子网 ==> 子网通过路由器与外部接洽 ==> 多个子网形成网络 ==> 路由器相当于网络中的一个节点。

    # Point of Attention

    • 集线器由于只能全广播，先已被交换机代替，交换机可指定性单播。

    • 集线器一般集成在路由器中。

    • 交换机既有全广播，也有单播功能。

      • 两台电脑通信，是需要先得到目标Mac地址的，而获取Mac地址的方式：利用ARP协议，先用全广播，在用单播，便可以获取到同一网络内的目标Mac地址。

        Mac地址是网卡的序列号，每个网卡有两个Mac地址，一个是唯一的，一个是通用的（用于全广播，即ff ff ff ff ff ff）

    • 交换机用于同一个网络内的电脑连接，而路由器用于两个网络之间的连接。后者比较范围大

      • 默认网关一般指的就是路由器，作用在于传递两个不同网络之间的数据。类似于关口。
      • 路由器最少要有两个网卡。因为要链接两个不同的网络。

    # IP地址的基本知识

    IP地址是一串32 bit的数字，即4个字节，每个字节为一组，分别用十进制表示，再用圆点分隔。

    IP地址 = 网络号 + 主机号，通过子网掩码/网络掩码来区分，如下图。子网掩码有两种表示方式；子网掩码分为两部分，左边部分全为1，右边部分全为0，全为1的是网络号，全为0的是主机号。

    # 为什么直接使用IP，而采用IP、域名并用

    • 域名不定长，处理复杂。
    • 域名一般都需要多于4个字节来表示，而IP只需要固定4个字节
    • IP不好记

    浏览器本身并不具备发送消息的功能，而是打包好，再调用函数使用System Call。

    浏览器向目标服务器发送HTTP请求消息之前，会先调用操作系统的Socket库中的DNS resolver向DNS服务器发起resolution，形式大致为gethostbyname www.xxx.com，然后DNS服务器会从域名与IP地址的对照表中查找相应的记录，并返回IP地址。

    # TCP

    • tcp的三次握手、四次挥手
      • 三次握手，建立连接，即connect，客户端一发一答，服务器一答（发含在里面），准备资源
      • 四次挥手，断开连接，即close，双方各有一次发一回，释放资源。---- 调用close的作用以及recv的反应
      • tcp通信结束的时候，都是客户端先调用close，留着服务器等待一段时间，服务器再关闭，这样可以确保最后发送的数据发送成功 ---- 客户端调用close之后也会等待2msl的时间
      • 先关闭的等 1msl，后关闭的等 2msl

    # URL

    URI，uniform resource identifier，统一资源标识符。URL，uniform resource locator，同一资源定位符，是 URI 的一种。

    形式

    scheme://host[:port]/path/.../[?query-string][#anchor]

    如，http://item.jd.com/11936238.html#product-detail

    说明

    形式中的被 [] 括起来的参数可省略不写。

    • scheme，协议，如 HTTP、HTTPS、FTP
    • host，服务器的 ip 地址或域名
    • port，服务器的端口，不写一般会使用默认端口，如 80 或 443
    • path，访问资源的路径
    • query-string，参数
    • anchor，锚点，如果一个 URL 带锚点，则访问的时候会定位到相应的位置

    Notice

    • URL包装进HTTP request header时，会进行URL编码，这是一种特定的编码，可以百度搜索进行在线解码

    # 请求方法

    HTTP 的请求方法有多种，GET、POST、OPTION、HEAD、PUT、DELETE

    GET

    POST

    • 安全性更高，用于传输重要的数据，例如用户名、密码、邮箱。
    • 一般用于表单提交数据，其他情况会使用 GET 请求。表单一般意味着数据是重要的。

    # 响应状态码

    • 2 开头，表示请求处理结束，且成功。如 200
    • 3 开头，表示请求正在处理，需跳转到其他 URL，以完成请求处理。如 302、307
    • 4 开头，表示客户端出错
    • 5 开头，表示服务器出错

    # HTTP 格式

    HTTP 作为一种网络通信协议，分为多种情况，当其发起请求时，一般只有请求头，格式为：请求行＋请求头字段

    常见字段

    • Host，主机和端口号
    • Connection，链接类型
    • Upgrade-Insecure-Requests，升级为 HTTPS 请求
    • User-Agent，浏览器型号、系统型号等
    • Accept，能够接收的文件类型
    • Referer，页面跳转处
    • Cookie
    • x-requested-with:XMLHttpRequest，表示为 Ajax 异步请求

    # HTTP 与 HTTPS

    HTTP

    • HTTP 协议是用于服务器与浏览器之间的数据传输协议，也叫超文本传输协议
    • 默认端口号 80
    • 明文方式传输，只能获取到，就能够直接读取。无认证。

    HTTPS

    • HTTP＋SSL，SSL 为安全套接字层
    • 默认端口 443
    • 加密方式传输，有认证
    • 比 HTTP 更安全，但性能偏低
