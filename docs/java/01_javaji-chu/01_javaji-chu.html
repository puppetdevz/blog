    # Java特点

    • 基于JVM虚拟机的跨平台语言，一次编写，到处运行
    • 易于编写，内置GC，不用考虑内存管理
    • JVM有工业级的稳定性和高度优化的性能，经过了长时间的考验
    • 最广泛的开源社区支持，各种高质量组件随时可用

    # Java简介

    Java最早由Sun公司（后被Oracle收购）的詹姆斯·高斯林开发；

    Java语言介于编译型语言和解释性语言之间，编译型语言如C、C++，代码直接编译成机器码，但是不同种类的CPU的指令集不同，所以需要针对不同CPU指令集编译出与之对应的机器码才能够跨平台执行。解释型语言，如Python、Ruby，则是直接由解释器加载源码运行，但是这样带来的缺点是运行速度较慢。

    Java是将代码编译成一种 字节码 ，这种字节码可以理解成机器码的伪代码，然后再使用Java虚拟机来加载运行字节码，所以Java的跨平台运行，其实是同一份代码，然后不同平台的Java虚拟机来运行而已，这样做，并不会怎么影响执行效率；这里需要注意的是，Java虚拟机是需要根据不同的平台（或者说不同类型的CPU指令集）来制作编写，官方制定了Java虚拟机的规范，并且这个规范现在被良好地执行着，所以Java的兼容性很好；

    # Java版本及其选择

    Java有三个版本：Java EE、Java SE、Java ME，分别是企业版、标准版、微型版；

    包含关系：Java EE > Java SE > Java ME；

    Java SE 包含标准的 JVM 和标准库，而 Java EE 是在 SE 版本上加了大量的 API 和库，以便其他的各种开发用途；Java ME主要针对嵌入式设备，与 SE、EE 并不兼容，目前并不流行，如无特殊需要，不建议学习；

    # Java参考学习路线

    1. Java SE 是 Java 平台的核心，先学习 Java SE，掌握 Java 语言本身、Java 核心开发技术以及 Java 标准库
    2. 进一步学习 Java EE，学习 Spring 框架、数据库开发和分布式架构
    3. 如果对大数据感兴趣，再进一步学习 Hadoop、Spark、Flink 大数据平台（这点最好对比Python，也许 Python 做大数据更好，或者两者结合）
    4. 如果转行移动开发，再深入 Android 平台

    # 专用名词

    • JRE：Java Runtime Environment；JRE本质是运行Java字节码的虚拟机，包含有JVM和Runtime Library
    • JDK：Java Development Kit；JDK包含JRE、Compiler和Debugger等等

    JDK 跟 JRE 的关系，简单讲，如果只有 Java 编译后的字节码，那使用 JRE 就可以直接运行了，但是需要是 Java 源码，则需要先编译成字节码，而JRE不包含编译器 Compiler 和 Debugger 等

    • JSR：Java Specification Request；Java 规范要求，如果想要给 Java 平台加功能，那么功能的编写必须满足这个规范；JVM 内存模型到 Web 程序接口都是按照这套规范标准化的

    • JCP 组织：Java Community Process，负责审核程序是否符合 JSP 规范的组织。

    • RI：Reference Implementation，参考实现；

      简单来讲，就是光有想法还不行，得要有能跑的代码，这个能跑的代码就是RI

    • TCK： Technology Compatibility Kit，技术兼容性测试套件；

      有了RI之后，如果有人有同样的想法，开发出了另一个具有同样功能的东西，那么就需要TCK来测试对方的接口以及功能是否相同

    对于 RI 和 TCK，一般不会用 RI，因为RI只是一个新奇的东西，各方面还不够稳定。

    • java：这个可执行程序其实就是 JVM，运行它，就是启动 JVM，然后让 JVM 执行指定的编译后的代码

    • jdb：Java 调试器，用于开发阶段的运行调试

    # Notice

    • Java 程序的缩进并不是必须的
    • Java 的区分大小写
    • Java 程序的文件名有限制：必须跟文件中的类名一致

      • 如果文件中有多个类，存在 public 类，则要与 public 类同名
      • 如果文件中只有一个类，则必须要跟这个唯一的类同名
    • Java 程序中，只允许存在一个或者没有 public 类
    • Java SE 11 之后，如果只是单个文件，且不引入第三方库，可以直接用 java xxx.java 运行程序，而不需要编译过程；但在实际开发中，很少不引入第三方库，所以这个特性很少用

    # double与float的区别

    float 是单精度类型，精度是 8 位有效数字，取值范围是 10^-38^ 到 10^38^，float 占用4个字节的存储空间。double 是双精度类型，精度是 17 位有效数字，取值范围是 10^-308^ 到 10^308^，double占用 8 个字节的存储空间

    当不声明的时候，默认小数都用 double 来表示，所以如果要用 float 的话，则应该在其后加上 f

    总之，double 的精度比 float 高得多，常用 double。

    开发过程中，涉及到金额的变量，都只会用 double，而不用 float。

    # 方法的重载

    多个方法的名称一样，但参数列表不一样。Java会根据传入的参数个数自动帮我们对应过去。

    参数列表不一样有：

    # 方法的参数-返回值

    • 所有的数据类型都可以作为参数传递给方法，也可以作为返回值返回，例如数组、自定义类、接口等等
    • 数组作为参数传递给方法时，传递过去的是其地址值。数组作为方法的返回值时，同样也是返回地址值。
    • 方法的参数数量可以是 0、1 或者多个。但方法的返回值，只能有一个。这跟 Python 不同，Python 可以有多个返回值。如果 Java 的方法想要返回多个值，可以利用数组。本质上，这个跟 Python 是一样的

    # Java程序的内存

    一个 Java 程序运行时，会向系统内存申请一块空间，但申请得到的空间并不是整块使用的，而是将其分成了五个部分。

    部分 用途
    栈（Stack） 只存放方法中的局部变量。方法的运行一定是在栈中。
    堆（Heap） 凡是 new 出来的东西，都在堆中。
    方法区（Method Area） 存储 .class 相关信息，主要包含方法的信息。
    本地方法区（Native Method Stack） 与操作系统相关
    寄存器（PC Register） 与 CPU 相关

    # Notice

    • Stack。局部变量是指方法的参数，或是方法 {} 内部定义的变量。变量一定超出该方法的作用域，就会从 Stack 中消失。

    # 类

    类与对象的关系：

    • 类是对一类事物的描述，是抽象的。
    • 对象是一类事物的实例，是具体的。
    • 类是对象的模板，对象是类的实体。

    1. **成员变量（属性）**写在类中，但在成员方法的外面。成员变量的类型可以为任何数据类型，如自定义类、接口等等

    2. 成员方法（行为）不能是 static，带有 static 一般是普通的方法。

    3. 使用某一个类之前，必须要进行导包，即告诉编译器要使用的类的位置。但是如果要使用的类在同一个包中，可以省略导包语句。此外，如果类在 java.lang 包中，也可以不写导包语句。

      需要注意的是，导包语句必须写在 package 语句之后，class 类之前。

    # 局部变量-成员变量

    局部变量跟成员变量是不一样的（成员变量：对比 Python，相当于实例属性）

    一共有 5 点不同

    • 定义位置不一样
      • 局部变量：定义在方法内部
      • 成员变量：定义在方法的外部，直接写在类当中
    • 作用范围不一样
      • 局部变量：只能在方法内部起作用，出了方法范围，无法使用
      • 成员变量：在类中通用
    • 默认值不一样
      • 局部变量：只声明，不赋值，则没有默认值，无法使用
      • 成员变量：只声明，不赋值，但 Java 虚拟机会赋默认值，可以使用
    • 内存位置不一样
      • 局部变量：在栈内存中；
      • 成员变量：在堆内存中
    • 生命周期不一样
      • 局部变量：随着方法进栈而生，出栈而灭
      • 成员变量：随着对象创建而生，对象被垃圾回收而灭

    # 继承中方法的覆盖重写

    重写(override)-重载(overload)

    • 重写，也叫覆盖重写，发生在继承中，方法名和参数列表都要相同

    • 重载，方法名相同，而参数列表不同。

    拓展：

    • java.lang.Object 是所有类的公共最高父类，任何类都继承于它，例如 java.lang.String 类也是 Object 的子类。
    • 权限修饰符：public > protected > (default) > private。（注意：(default)不是关键字 default，而是留空，不写权限修饰符。）
    • 重写的注意事项
      • @Override：写在方法之前，用来检测是否正确覆盖重写。只是一个注解，写与不写，只要重写正确，都没有问题。但是建议还是写上，因为 @Override 可以帮我们检测是否正确。
      • 子类重写的方法的返回值类型范围，要等于或小于父类方法的返回值类型，即如果父类的返回值类型为 String，那么子类重写的方法的返回值类型不能为 Object。
      • 子类重写的方法的权限必须要等于或大于父类方法的权限。即如果父类方法的权限为 (default)，那么子类重写的方法的权限至少也得是 (default)，或者为 protected、public 等。

    # 继承后的特点（构造方法）

    1. 子类构造方法中默认有隐含的 super() 调用，所以是先调用了父类的构造方法，再调用子类的构造方法。
    2. 子类构造方法可以通过 super() 手动调用父类的构造方法，并完成构造方法的重写。
    3. 父类构造方法 super() 在子类构造方法中的手动调用，必须是在构造方法的第一条语句。也就是说，一个子类的构造方法中，不能多次调用父类的构造方法 super()。

    总之，子类构造方法中必须调用父类的构造方法 super()，如果没有调用，则 JVM 会隐含帮我们添加上。

    # super三种用法

    1. 在子类的成员方法中，利用 super.父类成员变量 访问父类的成员变量。
    2. 在子类的成员方法中，利用 super.父类成员方法 访问父类的成员方法。
    3. 在子类的构造方法中，利用 super() 访问父类的构造方法。

    # this三种用法

    1. 在本类的成员方法中，访问本类的成员变量

    2. 在本类的成员方法中，访问本类的成员方法，当然，这种情况可不加 this，加了只是起到提示的作用。

    3. 在本类的构造方法中，访问本类的另一个构造方法。

      但是需要注意的是，this() 只能是构造方法中的 first statement，也就是说，super() 和 this() 不能够同时使用。

    # 抽象类-抽象方法注意事项

    关于抽象类的使用，以下为语法上要注意的细节，虽然条目较多，但若理解了抽象的本质，无需死记硬背。

    1. 抽象类不能创建对象，如果创建，编译无法通过而报错。只能创建其非抽象子类的对象。

      假设创建了抽象类的对象，调用抽象的方法，但抽象方法没有具体的方法体，没有意义。

    2. 抽象类中，可以有构造方法，是供子类创建对象时，初始化父类成员使用的。

      子类的构造方法中，有默认的 super()，需要访问父类构造方法。

    3. 抽象类中，不一定包含抽象方法，但是有抽象方法的类必定是抽象类

      未包含抽象方法的抽象类，目的就是不想让调用者创建该类对象，通常用于某些特殊的类结构设计。（只要是抽象类，就不可以直接用来创建对象）

    4. 抽象类的子类，必须重写抽象父类中所有的抽象方法，否则，编译无法通过而报错。除非该子类也是抽象 类。

      **假设不重写所有抽象方法，则类中可能包含抽象方法。**那么创建对象后，调用抽象的方法，没有意义。

    # 接口-默认方法

    从 Java 8 开始，接口中允许定义默认方法，其格式如下

    使用场景：解决接口升级的问题，举例

    现有一个接口，里面定义了一个抽象方法 abstractMethodA，有两个实现类对该接口进行实现，覆盖重写了该接口中所有的抽象方法。此时！接口需要进行升级，即再定义一些抽象方法，升级之后，前面的两个实现类就需要再次对接口中新定义的抽象方法进行覆盖重写，这会带来很大的麻烦，因为一旦出现遗漏，则会报错。所以，接口开始允许使用默认方法。

    • 默认方法大都是 public 权限，因此 public 可以省略不写，不写时默认为 public。但 default 不能省略。

    # equals()

    • public boolean equals(Object obj)：其他某个对象是否与此对象“相等”。这里的“相等”有默认和自定义两种情况。

    如果没有覆盖重写 equals()，Object 类中默认进行 == 运算符的对象地址比较，只要不是同一个对象，结果必然为 false

    如果希望进行对象的内容比较，即所有或指定的部分成员变量相同就判定两个对象相同，则可以覆盖重写 equals()

    import java.util.Objects;

public class Person {	
	private String name;
	private int age;
	
    @Override
    public boolean equals(Object o) {
        // 如果对象地址一样，则认为相同
        if (this == o)
            return true;
        // 如果参数为空，或者类型信息不一样，则认为不同
        if (o == null || getClass() != o.getClass())
            return false;
        // 转换为当前类型
        Person person = (Person) o;
        // 要求基本类型相等，并且将引用类型交给java.util.Objects类的equals静态方法取用结果
        return age == person.age && Objects.equals(name, person.name);
    }
}

    # Collection框架

    JavaSE 提供了满足各种需求的 API，在使用这些 API 前，先了解其继承与接口操作架构，才能了解何时采用哪个类，以及类之间如何彼此合作，从而达到灵活应用。

    集合按照其存储结构可以分为两大类，分别是单列集合 java.util.Collection 和双列集合 java.util.Map，今天我们主要学习 Collection 集合

    • Collection：单列集合类的根接口，用于存储一系列符合某种规则的元素，它有两个重要的子接口，分别是 java.util.List 和 java.util.Set。其中，List 的特点是元素有序、元素可重复。Set的特点是元素无序，而且不可重复。
    • List接口的主要实现类有 java.util.ArrayList 和 java.util.LinkedList ，Set接口的主要实现类有 java.util.HashSet 和 java.util.TreeSet

    Collection 是所有单列集合的父接口，因此在 Collection 中定义了单列集合 ( List 和 Set ) 通用的一些方法，这些方法可用于操作所有的单列集合。常用方法如下：

    • public boolean add(E e)： 把给定的对象添加到当前集合中 。
    • public void clear() :清空集合中所有的元素。
    • public boolean remove(E e): 把给定的对象在当前集合中删除。
    • public boolean contains(E e): 判断当前集合中是否包含给定的对象。
    • public boolean isEmpty(): 判断当前集合是否为空。
    • public int size(): 返回集合中元素的个数。
    • public Object[] toArray(): 把集合中的元素，存储到数组中。

    # Iterator接口

    在程序开发中，经常需要遍历集合中的所有元素。针对这种需求，JDK 专门提供了一个接口 java.util.Iterator。Iterator 接口也是 Java 集合中的一员，但它与 Collection、Map 接口有所不同，Collection 与 Map 接口用于存储元素，而 Iterator 用于迭代访问（即遍历）Collection 中的元素

    想要遍历 Collection 集合，那么就要获取该集合迭代器完成迭代操作，获取 Collection 集合对象的迭代器

    • public Iterator iterator(): 获取集合对应的迭代器，用来遍历集合中的元素的。

    Iterator 接口的常用方法如下：

    • public E next()：返回迭代的下一个元素。
    • public boolean hasNext()：如果仍有元素可以迭代，则返回 true。
    public class IteratorDemo {
  	public static void main(String[] args) {
        // 使用多态方式 创建对象
        Collection<String> coll = new ArrayList<String>();

        // 添加元素到集合
        coll.add("串串星人");
        coll.add("吐槽星人");
        coll.add("汪星人");
        //遍历
        //使用迭代器 遍历   每个集合对象都有自己的迭代器
        Iterator<String> it = coll.iterator();
        //  泛型指的是 迭代出 元素的数据类型
        while(it.hasNext()){ //判断是否有迭代元素
            String s = it.next();//获取迭代出的元素
            System.out.println(s);
        }
  	}
}

    # 泛型

    泛型，可以在类或方法中预知地使用未知的类型，同时使得代码能够复用。

    同时，泛型还有以下好处：

    • 将运行时期的 ClassCastException，转移到了编译时期变成了编译失败。
    • 避免了类型强转的麻烦。

    # 通配符高级使用（受限泛型）

    暂时了解即可，基本不用到

    设置泛型的时候，实际上是可以任意设置的，只要是类就可以设置。但是在 JAVA 泛型中可以指定一个泛型的 上限 和 下限

    泛型的上限：

    • 格式： 类型名称 <? extends 类 > 对象名称
    • 意义： 只能接收该类型及其子类

    泛型的下限：

    • 格式： 类型名称 <? super 类 > 对象名称
    • 意义： 只能接收该类型及其父类型

    比如：现已知 Object 类，String 类，Number ，Integer 类，其中 Number 是 Integer 的父类
