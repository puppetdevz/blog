    通过 jstat 可以对 jvm 堆的内存进行统计分析，而 jmap 可以获取到更加详细的内容

    如：内存使用情况的汇总、对内存溢出的定位与分析。

    jmap 的使用如下：

    C:\Users\puppet>jmap -h

    jmap -histo[:[<histo-options>]] <pid>

    # 将堆内存使用情况dump到文件

    有时候我们需要将jvm当前内存中的使用情况dump到文件中，然后对文件进行分析，jmap支持dump到文件中的。

    jmap -dump:<dump-options> <pid>

 dump-options:
      live         dump only live objects (takes precedence if both "live" and "all" are specified)
      all          dump all objects in the heap (default if one of "live" or "all" is not specified)
      format=b     binary format
      file=<file>  dump heap to <file>
      gz=<number>  If specified, the heap dump is written in gzipped format using the given compression level.
                   1 (recommended) is the fastest, 9 the strongest compression.

    Example: jmap -dump:live,format=b,file=heap.bin <pid>
    
# 示例
jmap -dump:live,format=b,file=heap.bin 19828

    # 通过jhat分析dump文件

    我们通过 jmap -dump 命令将jvm的堆内存使用情况dump到文件中，这个文件是一个二进制的文件，不方便查看，但是我们可以借助于jdk自带的jhat工具进行查看。

    # 用法： 
jhat ‐port <port> <file>
# 示例
jhat ‐port 9999 /tmp/dump.dat

    # 通过MAT工具分析dump文件

    MAT(Memory Analyzer Tool)，一个基于Eclipse的内存分析工具，是一个快速、功能丰富的JAVA heap分析工具，它可以帮助我们查找内存泄漏和减少内存消耗。（功能更加强大，也更加常用）

    使用内存分析工具从众多的对象中进行分析，快速的计算出在内存中对象的占用大小，看看是谁阻止了垃圾收集器的回收工作，并可以通过报表直观的查看到可能造成这种结果的对象。

    官网：Eclipse Memory Analyzer Open Source Project | The Eclipse Foundation (opens new window)

    从官网上下载后，直接绿色解压就算安装完成，打开 MemoryAnalyzer.exe 就算程序启动。

    # 大致使用

    打开 dump 文件（建议将文件拷贝到一个单独的目录后再导入，因为分析的时候会在项目所在目录下产生很多其他文件数据）

    选择打开模式，常用 Leak Suspects Report

    常用操作与关注要点：

    • 可疑报告中的 Problem Suspect
    • Actions 中的 Histogram：罗列出 jvm 中所有类对象实例（支持搜索）
    • Actions 中的 Dominator Tree：罗列出 jvm 中最大的活跃对象及所有对象的 Tree 图（支持搜索）

    # 实战：堆内存溢出的分析与定位

    内存溢出在实际的生产环境中经常会遇到，比如，不断的将数据写入到一个集合中、出现了死循环、读取超大的文件等等，都可能会造成内存溢出。

    如果出现了内存溢出，我们需要定位到发生内存溢出的环节，然后进行分析，判断该溢出是正常还是非正常情况，如果是正常的需求，就应该考虑加大内存的设置，如果是非正常需求，那么就要对代码进行修改，修复这个bug。

    那么如何定位问题呢？我们需要借助于jmap与MAT工具。

    下面我们进行实战模拟

    # 模拟内存溢出

    向List集合中添加100万个字符串，每个字符串由1000个UUID组成。如果程序能够正常执行，最后打印“完成”

    public class DemoJvmOutOfMemory {
    public static void main(String[] args) {
        ArrayList<String> strings = new ArrayList<>();
        for (int i = 0; i < 1000000; i++) {
            StringBuilder stringBuilder = new StringBuilder();
            for (int j = 0; j < 1000; j++) {
                stringBuilder.append(UUID.randomUUID());
            }
            strings.add(stringBuilder.toString());
        }
        System.out.println("完成");
    }
}
