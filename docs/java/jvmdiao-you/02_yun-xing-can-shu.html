    本文基于 Java 17

    JVM 的参数类型可以分为三类：

    • 标准参数：-help、-version
    • -X 参数（非标准参数）：-Xint、-Xcomp
    • -XX 参数（使用率较高，也是非标准参数，常用于 JVM 调优或调试）：-XX:newSize、-XX:+UseSerialIGC

    # 标准参数

    标准参数，一般都是很稳定的，在未来的 JVM 版本中不会改变，可以使用 java -help 检索出所有的标准参数。

    C:\Users\puppet>java -help

           （执行类）
   或  java [options] -jar <jar 文件> [args...]
           （执行 jar 文件）
   或  java [options] -m <模块>[/<主类>] [args...]
       java [options] --module <模块>[/<主类>] [args...]
           （执行模块中的主类）
   或  java [options] <源文件> [args]
           （执行单个源文件程序）

 将主类、源文件、-jar <jar 文件>、-m 或
 --module <模块>/<主类> 后的参数作为参数
 传递到主类。

    # -server与-client

    这两个参数在 jdk 17 中，通过 java --help 已经没有找到（实际上也存在，只是没有列出，也说明不用该参数了），但在 jdk 8 中还存在。简单了解一下

    -server 或 - client 用来设置 jvm 的运行参数，指定 VM 采用 server 类型还是 client 类型，两者的区别：

    • Server VM：初始化堆空间较大，默认使用并行垃圾回收器，启动速度慢，运行速度快。
    • Client VM：初始化堆空间较小，默认使用串行垃圾回收器，启动速度快，运行速度慢。

    但是 jvm 在启动的时候会根据计算机的硬件和操作系统自动选择使用 Server VM 还是 Client VM：

    • 如果操作系统为 32 位
      • 如果是 Windows 系统，无论硬件配置如何，均默认采用 Client VM
      • 如果是其他操作系统，机器配置有 2GB 以上内存，且有两个以上 CPU，则默认采用 Server VM，否则采用 Client VM
    • 如果操作系统为 64 位：只有 Server VM，不支持 Client VM
    D:\IdeaProjects\demo-jvm\src>java -client --show-version DemoJVM
java 17.0.2 2022-01-18 LTS
Java(TM) SE Runtime Environment (build 17.0.2+8-LTS-86)
Java HotSpot(TM) 64-Bit Server VM (build 17.0.2+8-LTS-86, mixed mode, sharing)
defaultParam

    # -Xint、-Xcomp和-Xmixed

    这三个参数可以设置 jvm 对字节码的执行模式

    • -Xint：将 jvm 设置为解释模式。

      在解释模式（interpreted mode）下，jvm 会强制执行所有字节码，而这会降低运行速度，通常低 10 倍或更多。

    • -Xcomp：将 jvm 设置为编译模式

      在编译模式（compiled mode）下，JVM 在第一次调用方法时，会把所有的字节码编译成本地代码，从而带来最大程度的优化。（与解释模式相反）

      但是，很多应用处于编译模式时也会有一些性能损失（虽然比解释模式损失的少），因为编译模式下，JVM 并没有启用 JIT 编译器的全部功能。（JIT 编译器可以判断字节码是否需要进行编译，并不是所有字节码都需要进行编译，对于一些只执行一次的字节码就不需要，如果编译了，则会浪费性能）

    • -Xmixed：将 jvm 设置为混合模式

      用大白话讲，就是将解释模式与编译模式进行混合使用，由 jvm 自己决定，这是 jvm 默认的模式，也是推荐使用的模式。

    D:\IdeaProjects\demo-jvm\src>java --show-version -Xint DemoJVM
java 17.0.2 2022-01-18 LTS
Java(TM) SE Runtime Environment (build 17.0.2+8-LTS-86)
Java HotSpot(TM) 64-Bit Server VM (build 17.0.2+8-LTS-86, interpreted mode, sharing)
defaultParam
# 说明：interpreted mode 第一次运行比较快

D:\IdeaProjects\demo-jvm\src>java --show-version -Xcomp DemoJVM
java 17.0.2 2022-01-18 LTS
Java(TM) SE Runtime Environment (build 17.0.2+8-LTS-86)

defaultParam

    # -XX参数

    -XX 参数也是非标准参数，主要用于 jvm 的调优和 debug 操作。

    -XX 参数的使用有 2 种方式，一种是 boolean 类型，一种是非 boolean 类型：

    • boolean 类型：-XX:[+/-]

      如：-XX:+DisableExplicitGC

      表示禁用手动调用 gc 操作，也就是说在代码中调用 System.gc() 无效（如果不希望程序员在代码中手动调用 System.gc() 去进行垃圾回收，则可以使用该参数）

    • 非 boolean 类型：-XX:

      如：-XX:NewRatio=1 表示新生代和老年代的比值

    # -Xms与-Xmx

    这两个参数比较特殊，属于 -XX 参数

    -Xms 与 - Xmx 分别是设置 jvm 的堆内存的初始大小和最大大小。

    • -Xms512m：等价于 -XX:InitialHeapSize，设置 JVM 初始堆内存为 512M。
    • -Xmx2048m：等价于 -XX:MaxHeapSize，设置 JVM 最大堆内存为 2048M。

    适当的调整 jvm 的内存大小，可以充分利用服务器资源，让程序跑的更快。

    D:\IdeaProjects\demo-jvm\src>java -Xms64m -Xmx256m DemoJVM

    # 查看正在运行的程序的jvm运行参数

    如果想要查看正在运行的程序的 jvm 参数就需要借助 jinfo 命令查看

    # 查看所有的参数
jinfo -flags <进程id>

# 查看某一参数的值
jinfo -flag <参数名> <进程id>
# 例如：
jinfo -flag MaxHeapSize 6219
